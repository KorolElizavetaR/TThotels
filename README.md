Стек: Maven, Java 21, Spring Boot, Spring JPA, Liquibase, H2
+ Swagger документация (не шибко подробная но присутствует)
Запускается с mvn spring-boot:run и пашет на 8092 порту

## Примечания по эндпоинтам

1. **GET /hotels** : В текущей реализации делает 3 запроса из-за lazy-фетчинга, одним энтити графом можно было бы поправить - оставила пока что как есть

2. **GET /hotels/{id}** : а вот тут как раз засчет энтити графа все достается одним запросом 

3. **GET /search** : та же проблема что и GET /hotels. Метод принимает сколь угодно параметров поиска и их можно комбинировать. `?amenities={amenities1}&amenities={amenities2}&city={cityname}`
   - amenities - может быть массивом, то бишь искать отели по нескольким удобствам

4. **POST /hotels/{id}/amenities** : на повторный запрос будет перезаписывать все 'удобства' отеля, сохраняя существующие. Из плюсов: эндпоинт хорошо отработает в многопоточной среде - предусмотрена гонка состояний. Из плохого - много запросов к бд. Можно было бы сначала достать все сущности из бд, а то, что не досталось - прогонять через findOrCreateByName.
